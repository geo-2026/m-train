# M-TRAIN 저장소(Google Apps Script) 배포 안내

> 이 단계는 **선생님 구글 계정(geoaikakoki@gmail.com)으로 직접** 진행합니다.
> **비밀번호를 누구에게도 알려줄 필요가 없습니다.** 결과로 나오는 `.../exec` 주소 한 줄만 있으면 됩니다.

학생 질문·교실 대표 대화를 보관할 "저장소"를 선생님 구글 계정 안에 만드는 과정이에요. 한 번만 하면 됩니다.

## 1) Apps Script 프로젝트 만들기
1. `geoaikakoki@gmail.com` 으로 로그인한 상태에서 **https://script.google.com** 접속
2. 좌측 상단 **`+ 새 프로젝트`** 클릭
3. 기본으로 열린 `Code.gs`(또는 `코드.gs`) 내용을 **모두 지우고**, 이 폴더의 **`Code.gs` 내용을 통째로 붙여넣기**
4. 상단 💾(저장) 클릭 — 프로젝트 이름은 아무거나(예: `M-TRAIN 저장소`)

## 2) 웹앱으로 배포하기
1. 오른쪽 위 **`배포` → `새 배포`**
2. ⚙️(톱니) → **`웹 앱`** 선택
3. 설정:
   - **설명**: M-TRAIN (아무거나)
   - **다음 사용자로 실행**: **나 (geoaikakoki@gmail.com)**
   - **액세스 권한이 있는 사용자**: **모든 사용자**  ← (중요! 학생들이 로그인 없이 쓰게 함)
4. **`배포`** 클릭
5. 처음이면 **권한 검토/승인** 창이 떠요 →
   - 계정 선택 → "이 앱은 Google에서 확인하지 않았습니다" 화면이면 **`고급` → `(프로젝트 이름)(으)로 이동`** → **`허용`**
   - (선생님 본인이 만든 스크립트라 안전합니다. 시트를 만들고 읽고 쓰는 권한이에요.)
6. 배포 완료 화면의 **`웹 앱` URL**을 복사합니다. 형태:
   ```
   https://script.google.com/macros/s/AKfycb.................../exec
   ```

## 3) 이 URL을 앱에 연결
- 복사한 `.../exec` 주소를 **저(클로드)에게 알려주시면** 제가 앱(`docs/js/config.js`)에 넣고 다시 배포합니다.
- 직접 하실 경우: `docs/js/config.js` 의 `DEFAULT_APPS_SCRIPT_URL = ''` 에 주소를 넣고 저장 → 깃허브에 푸시.

## 확인
- 브라우저에서 그 `.../exec` 주소를 그냥 열어보면 `{"ok":true,"app":"M-TRAIN", ...}` 같은 글자가 보이면 정상입니다.
- 선생님 구글 드라이브에 **`M-TRAIN 데이터 (자동 생성)`** 라는 시트가 자동으로 생깁니다(질문·대화가 여기 임시 저장됨).

## 자주 묻는 것
- **학생도 로그인해야 하나요?** 아니요. 학생은 `geo-2026.github.io/M-TRAIN/` 주소만 열면 됩니다.
- **개인정보는요?** 학번·이름은 시트에 저장되지 않습니다(학생 단말기 + 본인 PDF에만). 시트에는 *익명 질문*과 *교실 대표 대화*만 남습니다.
- **데이터 삭제는?** 교사 화면(10페이지)의 **"수업 종료 — 우리 반 질문·대화 모두 삭제"** 버튼으로 학급별로 비웁니다.
- **코드를 고치면?** Apps Script에서 `배포 → 배포 관리 → (연필) → 새 버전 → 배포` 하면 **같은 URL이 유지**됩니다.
